# مشکلات برنامه — استخراج‌شده از مطالعه کد فلاتر و بک‌اند
**تاریخ:** 2026-07-18 | **مبنا:** ۳۷۷ فایل Dart (~۵۳٬۰۰۰ خط) + بک‌اند Cloudflare Workers (~۶٬۵۰۰ خط TS، ۲۵ Migration)

## الف) مشکلات بحرانی (باید قبل از انتشار حل شوند)

1. **پشتیبانی آفلاین اصلاً پیاده نشده.** پوشه `lib/core/sync/` خالی است؛ نه صف رویداد محلی (SQLite)، نه `client_event_id`، نه Batch Sync، نه Cache درس‌ها. بخش ۲۲ سند (که آفلاین را «حالت عادی» دانش‌آموز افغان می‌داند) کاملاً غایب است. حتی فونت‌ها هم از `google_fonts` (نیازمند اینترنت) لود می‌شوند — بخش فونت آفلاین در pubspec کامنت است.
2. **جدول `audit_logs` وجود ندارد.** اصل Auditability (بخش ۱.۲ و ۲۰.۳ سند): هیچ Migration و هیچ کدی برای ثبت اقدامات Admin، Override نمره، یا فراخوانی AI با Prompt کامل نیست. فقط `safety_events` و `ai_teacher_chat_logs`/`answer_logs` جزئی پوشش می‌دهند.
3. **«آکادمی» یک منبع حقیقت موازی و پرخطر است.** `AcademyStore` یک Singleton درون‌حافظه‌ای همگام با شاگردان نمونه ثابت است که در ده‌ها صفحه مستقیم استفاده می‌شود (نقض Clean Architecture و اصل بخش ۴)؛ با «امتحانات»، «کتابخانه» و «CMS» سرورمحور هم‌پوشانی دارد. سند `SERVER-CONNECTIONS.md` خودش هشدار داده. باید با سیستم امتحانات یکپارچه شود.
   > **به‌روزرسانی ۲۳ جولای:** گام اول ادغام انجام شد — نمره‌دهی سمت سرور (چهارگزینه‌ای/صحیح‌غلط قطعی + تشریحی با همان AI امتحانات) و هماهنگ‌سازی مجوز والد. **ادغام معماری کامل (یک منبع حقیقتِ واحد به‌جای دو جدول/Store موازی) هنوز انجام نشده** — این بند همچنان باز است.
4. **Rate Limiting وجود ندارد.** هیچ محدودیت نرخی روی `/auth/login` و `/auth/register` در کد Worker نیست — برخلاف بخش ۲۰.۸ و ۳ب.۲ سند؛ حدس Brute-force کد دعوت و رمز عملاً آزاد است.
5. **CORS کاملاً باز است:** `ALLOWED_ORIGIN = "*"` در `wrangler.toml`.
6. **شناسه‌های حساس در مخزن Git کامیت شده‌اند:** `CF_ACCOUNT_ID`، `database_id` دیتابیس D1، کد مشتری Stream در `wrangler.toml` — و سند SPEC هم Tunnel/Connector ID دارد. مخزن نباید Public شود یا این مقادیر باید به Secret منتقل شوند.
7. **رمزگذاری ستونی AES-256 پیاده نشده.** بخش ۲۰.۲/۳ب.۵ سند: شماره تلفن، تاریخ تولد، کدهای دعوت در D1 به‌صورت متن ساده ذخیره می‌شوند.

## ب) انحرافات معماری از سند (سند باید آپدیت شود یا کد اصلاح شود)

8. **Stack بک‌اند کاملاً متفاوت است:** سند می‌گوید NestJS + PostgreSQL (pgvector) + Redis + MinIO روی سرور محلی؛ واقعیت: **Cloudflare Workers (Hono) + D1 (SQLite) + R2**. هیچ Redis/WebSocket/pgvector وجود ندارد.
9. **Schema واقعی D1 با Schema بخش ۱۷ سند تطابق ندارد:** نه `roles/permissions` جدول‌محور (نقش رشته‌ای در `users`)، نه `student_progress` پلی‌مورفیک (به‌جایش `student_lesson_views`، `student_chapter_completions`)، نه `grading_policies` (آستانه‌ها احتمالاً hard-coded)، نه `grades_scores`. در عوض `exam_attempts` که سند حذفش کرده بود، وجود دارد.
10. **RAG با معماری متفاوت:** به‌جای pgvector و `knowledge_collections` جدا per مضمون+صنف، embedding ها (`text-embedding-3-small`) در جدول `lesson_embeddings` در D1 و شباهت کسینوسی در خود Worker محاسبه می‌شود — بدون تفکیک معماری Collection ها (تضمین «هرگز از مضمون دیگر» ضعیف‌تر است) و با محدودیت مقیاس.
11. **تأیید ایمیل اضافه شده** (migration 0013، `verify-email`/`resend-verification`، بنر تأیید در اپ) — در حالی که سند v2.3 می‌گوید حساب با Invite Code «بلافاصله active، بدون هیچ مرحله تأیید». تناقض مستقیم؛ سند باید این را رسمی کند.
12. **PBKDF2 به‌جای bcrypt/argon2** (توجیه فنی معتبر: Workers ماژول Native ندارد) — سند باید به‌روز شود؛ ۱۰۰هزار تکرار فعلی قابل قبول ولی حداقل مستندسازی لازم است.
13. **چت Real-time نیست:** WebSocket در Workers پیاده نشده؛ چت با Polling کار می‌کند. بخش ۱۰.۳ سند (WSS/Socket.io) برآورده نیست. Push Notification (FCM/APNs) هم وجود ندارد — فقط اعلان درون‌اپی.
14. **نقش Instructor مسیر ثبت‌نام مستقیم دارد** (`/register/instructor` + invite store مخصوص) — در سند هیچ جریان ثبت‌نام Instructor تعریف نشده.
15. **قابلیت‌های کاملاً جدید خارج از سند** (باید وارد سند شوند): مشاور هوشمند (Advisor)، پلان مطالعه (Study Plan)، حافظه جمعی/فید اجتماعی (Collective Memory)، گیمیفیکیشن (points_ledger/levels)، پیام صوتی چت، TTS/STT صدای معلم (Azure)، پخش زنده سمینار (Cloudflare Stream + Jitsi + اتاق داخلی)، آکادمی، رضایت‌نامه (consents)، آواتار/عکس پروفایل روی R2، بازیابی رمز با ایمیل.

## ج) مشکلات کیفیت کد و نگه‌داری

16. **۲۰ فایل Mock DataSource هنوز موجودند** با سوییچ سراسری `kUseLiveBackend` — خطر انحراف (drift) بین رفتار Mock و Live؛ برخی ماژول‌های Admin (system_health، reports و...) هنوز فقط Mock دارند و باید تک‌تک تأیید شوند.
17. **تست تقریباً وجود ندارد:** پوشه `test/` از ۳ جولای دست‌نخورده است؛ برای ۵۳هزار خط کد، پوشش تست نزدیک صفر است (معیار پذیرش فاز ۱۰ سند برآورده نمی‌شود).
18. **اسناد داخلی مخزن کهنه‌اند:** `FINAL_PROJECT_REPORT.txt` (۵ جولای، UTF-16، قبل از ساخت بک‌اند)، `README.md` و description در `pubspec.yaml` هنوز می‌گویند «Phase 1: UI Prototype with mock data».
19. **Jitsi فقط Android/iOS** — فال‌بک ویدیوکنفرانس سمینار روی Web کار نمی‌کند (در pubspec ذکر شده ولی رفتار Web باید تعریف شود).
20. **`AZURE_TTS_VOICE` پیشنهادی در کامنت‌ها (`prs-AF-FatimaNeural`) قبلاً نامعتبر بوده** (کامیت e7d89a7 «invalid Azure voice» را فیکس کرده) — تنظیم صدا شکننده است و مستند رسمی ندارد.
21. **دو سیستم کد دعوت موازی در کلاینت:** `student_invite_store`، `instructor_invite_store`، `guardian_link_store` در `core/` جدا از feature auth — پراکندگی منبع حقیقت.

## د) موارد نیازمند تأیید/تصمیم صاحب پروژه

22. منطق کامل قفل صنف C1–C6 (بخش ۶ سند) در `lib/progress.ts` بک‌اند (۳۴۶ خط) ساده‌سازی شده — کدام آستانه‌ها واقعاً اجرا می‌شوند باید ممیزی و با سند هم‌راستا شود.
23. وزن‌های نمره‌دهی (۱۵/۱۵/۳۰/۴۰) پارامتری نیستند؛ اگر Admin باید بتواند تغییر دهد، جدول تنظیمات لازم است.
24. سیاست نگه‌داری/ناشناس‌سازی گفتگوهای AI پس از ۱۸۰ روز (بخش ۱۰.۴.۵) پیاده نشده.
25. دامنه در سند SPEC غلط است: `afghanistangriddigitalschool.org` ← صحیح طبق کد: **`afghanistangirlsdigitalschool.org`** (BASE_URL: `api.afghanistangirlsdigitalschool.org/api/v1`).

## و) یافته‌های مرور داشبورد Cloudflare (2026-07-18)

26. **مسیرهای Tunnel سایه‌زده شده‌اند (ناهماهنگی پیکربندی):** تونل برای `api.…` → `localhost:8080` و دامنه اصلی → `localhost:3000` مسیر دارد، ولی DNS همین دو نام را به **Worker** و **Pages** داده است؛ ترافیک هرگز به تونل نمی‌رسد. باید تصمیم گرفته شود: مسیرهای تونل حذف/بازتعریف شوند یا مستند شود که فقط برای آینده (سرور محلی AI) رزرو هستند.
27. **`ALLOWED_ORIGIN=*` در Production تأیید شد** (همان مشکل ۵) — در تنظیمات زنده Worker هم Plaintext است.
28. **Observability/Logs در Worker خاموش است** — هیچ لاگ ماندگاری برای عیب‌یابی Production وجود ندارد (روشن‌کردن آن رایگان و بدون تغییر کد است).
29. **هیچ Cron Trigger تعریف نشده** — کارهای زمان‌بندی‌شده (ناشناس‌سازی ۱۸۰روزه، پاک‌سازی `refresh_tokens`/`email_tokens` منقضی، محاسبات شبانه) جایی برای اجرا ندارند.
30. **پلن Free Workers (سقف ۱۰۰هزار درخواست/روز)** — با رشد کاربران باید پایش شود؛ مصرف فعلی ~۱٪ است.
31. **وضعیت Stream نامشخص:** کد مشتری و توکن تنظیم شده ولی صفحه Live Inputs خالی است — پخش زنده سمینار باید یک‌بار تست واقعی شود (در نبود Stream، اپ طبق طراحی به Jitsi/لینک دستی برمی‌گردد).
32. **رکورد `www` وجود ندارد** (هشدار خود Cloudflare) — بازدید `www.afghanistangirlsdigitalschool.org` شکست می‌خورد؛ یک CNAME + Redirect کافی است.
33. **DMARC تنظیم نشده** (هشدار خود Cloudflare) — برای جلوگیری از جعل ایمیل از دامنه مدرسه، افزودن رکورد DMARC لازم است (SPF/DKIM موجودند).
34. **Worker روی آدرس عمومی `…workers.dev` هم باز است** — در صورت تمایل می‌توان آن را خاموش کرد تا تنها مسیر رسمی `api.…` باشد.
35. **ارسال ایمیل با Resend انجام می‌شود** (`RESEND_API_KEY` + DKIM resend) و `GROQ_API_KEY` نیز به‌عنوان Provider جایگزین تعریف شده — هیچ‌کدام در اسناد قبلی مستند نبودند.
