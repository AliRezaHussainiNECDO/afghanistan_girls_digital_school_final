# راه‌اندازی: تأیید ایمیل، بازیابی پسورد، و عکس پروفایل

این سند مراحل لازم برای فعال‌کردن قابلیت‌های جدید را شرح می‌دهد. بعد از انجام
این مراحل، اپ به‌جای دادهٔ محلی موقت، به سرور واقعی وصل می‌شود و حساب‌ها در
دیتابیس D1 پایدار می‌مانند.

## چه چیزهایی اضافه شد

- **تأیید ایمیل هنگام ثبت‌نام:** پس از ثبت‌نام، لینک تأیید به ایمیل کاربر
  فرستاده می‌شود. ورود آزاد است، اما تا وقتی ایمیل تأیید نشده، در بالای
  داشبورد بنر «ایمیل خود را تأیید کنید» با دکمهٔ «ارسال مجدد لینک» دیده می‌شود.
- **بازیابی پسورد با کد ۶ رقمی:** کاربر ایمیلش را وارد می‌کند، کد ۶ رقمی به
  ایمیل می‌رود، و در خود اپ کد + رمز جدید را وارد می‌کند.
- **گزینهٔ «مرا به خاطر بسپار»** در صفحهٔ ورود (ذخیرهٔ ایمیل/رمز به‌صورت محلی).
- **عکس پروفایل سراسری:** عکس روی سرور (R2) آپلود می‌شود و در همهٔ بخش‌هایی که
  معلومات کاربر نمایش داده می‌شود (داشبورد، منوی کناری، لیست کاربران مدیر،
  لیست هم‌صنفی‌ها در چت) دیده می‌شود — نه فقط صفحهٔ پروفایل.

## علت باگ لاگین (رفع شد)

اپ به‌صورت پیش‌فرض روی حالت «داده محلی موقت» (`kUseLiveBackend = false`) بود.
در این حالت حساب ثبت‌نام‌شده فقط در حافظهٔ اپ نگهداری می‌شد؛ پس از بستن اپ،
حساب از بین می‌رفت و ورود دوباره با پیام «ایمیل یا رمز را چک کنید» شکست
می‌خورد (با اینکه ایمیل و رمز درست بود). اکنون پیش‌فرض به `true` تغییر کرد تا
حساب‌ها واقعاً در دیتابیس سرور ساخته و پایدار شوند. همچنین نشست کاربر هنگام
باز شدن اپ به‌صورت خودکار از روی Refresh Token بازیابی می‌شود.

## مرحله ۱ — اعمال Migrationها روی دیتابیس

```bash
cd backend

# اگر قبلاً حساب مدیر کل را نساخته‌اید:
npx wrangler d1 execute afghan_girls_school_db --remote --file=./migrations/0012_super_admin.sql

# ستون‌های جدید (email_verified, avatar_url) + جدول email_tokens:
npx wrangler d1 execute afghan_girls_school_db --remote --file=./migrations/0013_email_verification.sql
```

> نکته: کاربران موجود به‌صورت خودکار «تأییدشده» علامت می‌خورند تا دچار
> محدودیت نشوند.

## مرحله ۲ — تنظیم سرویس ایمیل (Resend)

بک‌اند روی Cloudflare Workers است و نمی‌تواند مستقیم SMTP بفرستد؛ از
[Resend](https://resend.com) استفاده می‌کنیم (پلن رایگان ۱۰۰ ایمیل/روز).

1. در resend.com ثبت‌نام کنید.
2. دامنهٔ `afghanistangirlsdigitalschool.org` را در بخش Domains اضافه و
   با رکوردهای DNS تأیید (Verify) کنید.
3. یک API Key بسازید و آن را به‌صورت راز روی Worker بگذارید:

```bash
npx wrangler secret put RESEND_API_KEY
# سپس کلید را بچسبانید
```

4. (اختیاری) اگر می‌خواهید نام فرستنده را تغییر دهید، در `wrangler.toml`
   زیر `[vars]` این را بگذارید:

```
EMAIL_FROM = "مکتب دیجیتال دختران افغانستان <no-reply@afghanistangirlsdigitalschool.org>"
```

> **بدون کلید هم کار می‌کند:** اگر `RESEND_API_KEY` تنظیم نشود، ایمیل واقعی
> فرستاده نمی‌شود و متن لینک/کد فقط در لاگ Worker چاپ می‌شود
> (`npx wrangler tail`) — برای تست کافی است، اما برای کاربران واقعی حتماً
> کلید را بگذارید.

## مرحله ۳ — استقرار مجدد Worker

```bash
cd backend
npx wrangler deploy
```

## مرحله ۴ — ساخت مجدد اپ

اپ اکنون به‌صورت پیش‌فرض به سرور واقعی وصل می‌شود؛ فقط دوباره Build کنید:

```bash
flutter build apk        # یا اجرای معمولی: flutter run
```

اگر برای تست موقت خواستید حالت محلی را برگردانید:

```bash
flutter run --dart-define=USE_LIVE_BACKEND=false
```

## نکات امنیتی پیاده‌سازی‌شده

- توکن لینک تأیید و کد ۶ رقمی فقط به‌صورت **هش SHA-256** در دیتابیس ذخیره
  می‌شوند (نه متن خام).
- پاسخ «فراموشی رمز» و «ارسال مجدد» همیشه یکسان است تا وجود/عدم‌وجود ایمیل
  لو نرود (ضد User Enumeration).
- کد بازیابی حداکثر ۵ تلاش و ۱۵ دقیقه اعتبار دارد؛ لینک تأیید ۴۸ ساعت.
- پس از تغییر رمز، همهٔ نشست‌های قبلی باطل می‌شوند.
- عکس پروفایل حداکثر ۵ مگابایت و فقط فایل تصویری پذیرفته می‌شود.
