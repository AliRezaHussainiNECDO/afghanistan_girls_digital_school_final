# مدیریت رازها (Secrets) — راهنمای کوتاه

این سند به بند «شناسه‌های حساس در Git» (docs/02، بند ۶) پاسخ می‌دهد. جمع‌بندی
بعد از بررسی کد واقعی: **هیچ رازِ واقعی (کلید API، توکن، کلید خصوصی) در
`wrangler.toml` یا هیچ فایل دیگرِ کامیت‌شده نیست** — همهٔ آن‌ها از قبل با
`wrangler secret put` تنظیم می‌شوند (کامنت‌های خودِ `wrangler.toml` این را
مستند کرده‌اند). آنچه واقعاً در `wrangler.toml` هست، دو نوع مقدار غیرمحرمانه
است:

## چرا `CF_ACCOUNT_ID` / `database_id` / `CF_STREAM_CUSTOMER` مشکل امنیتی نیستند

این‌ها **شناسه** هستند، نه **اعتبارنامه** (Credential). داشتنِ صرفِ شناسهٔ
حساب Cloudflare یا شناسهٔ یک دیتابیس D1 به کسی اجازهٔ هیچ کاری نمی‌دهد — بدون
یک API Token واقعی (که هرگز در این مخزن نیست)، این مقادیر مثل داشتن شمارهٔ
حساب بانکی بدون رمز/کارت است: قابل مشاهده، غیرقابل سوءاستفاده. خودِ مستندات
و قالب‌های رسمی Cloudflare هم همین الگو را دنبال می‌کنند (`database_id` در
`wrangler.toml` نمونه‌های رسمی Cloudflare هم کامیت می‌شود). بنابراین این مورد
از فهرست «ریسک بحرانی» به یک نکتهٔ بهداشتیِ اختیاری تنزل داده شد؛ اگر
می‌خواهید احتیاط بیشتری اعمال شود، می‌توان این‌ها را هم به Secret منتقل کرد،
اما این تغییری در سطح واقعیِ ریسک ایجاد نمی‌کند.

## رازهای واقعی — فهرست کامل (همه از قبل با `wrangler secret put`)

| نام | برای چه | در نبودش چه می‌شود |
|---|---|---|
| `JWT_SECRET` | امضای توکن ورود | **اجباری** — بدون آن هیچ‌کس نمی‌تواند وارد شود |
| `AI_PROVIDER_KEY` | معلم هوشمند، تولید سؤال، نمره‌دهی تشریحی | آن قابلیت‌ها غیرفعال، بقیه سالم |
| `AZURE_TTS_KEY` | صدای معلم هوشمند | صدا غیرفعال، متن سالم |
| `GEMINI_API_KEY` | نمره‌دهی خودکار مشق (Vision) | آپلود مشق کار می‌کند، نمره‌دهی خودکار نه |
| `CF_STREAM_TOKEN` | پخش زندهٔ سمینار روی Stream | فال‌بک خودکار به Jitsi/لینک دستی |
| `FCM_CLIENT_EMAIL` / `FCM_PRIVATE_KEY` | Push واقعی روی گوشی | اعلان فقط داخل‌اپی می‌ماند |
| `RESEND_API_KEY` | ارسال ایمیل (تأیید حساب، بازیابی رمز) | ایمیل ارسال نمی‌شود |
| `GROQ_API_KEY` | ارائه‌دهندهٔ جایگزین LLM | فقط اگر به‌عنوان Fallback پیکربندی شده باشد |
| `CERT_SIGNING_PRIVATE_KEY` | امضای رمزنگاری‌شدهٔ گواهی‌نامه | گواهی بدون امضا صادر می‌شود (نشان ⚠️) |
| `COLUMN_ENCRYPTION_KEY` | رمزگذاری تلفن/تاریخ تولد (تازه — این نشست) | ذخیره‌سازی Fail-safe به متن‌سادهٔ قبلی برمی‌گردد |

## تنظیم برای توسعهٔ محلی

```
cp .dev.vars.example .dev.vars
# مقادیر واقعی (یا خالی برای غیرفعال‌گذاشتن یک قابلیت) را در .dev.vars بگذارید
```

`.dev.vars` از قبل در `.gitignore` است — تأیید شد که هرگز کامیت نشده.

## تنظیم برای Production

```
wrangler secret put JWT_SECRET
wrangler secret put COLUMN_ENCRYPTION_KEY   # openssl rand -base64 32 | wrangler secret put COLUMN_ENCRYPTION_KEY
# ... و بقیه طبق جدول بالا، هرکدام را که فعال می‌کنید
```
