# اتصال بخش‌های باقی‌مانده به سرور — راهنمای استقرار

این سند مکملِ `SETUP-EMAIL-AND-AVATAR.md` است و بخش‌هایی را که تازه به سرور و
دیتابیس وصل شدند پوشش می‌دهد.

## چه چیزهایی وصل شد

### ۱) مدیریت تفصیلی شاگرد (پنل مدیر)
قبلاً همیشه روی دادهٔ محلی (Mock) بود. حالا به سرور وصل است:
- لیست صفحه‌بندی‌شدهٔ شاگردان با فیلتر (صنف، ولایت، وضعیت، در معرض خطر).
- جزئیات کامل تحصیلی (پیشرفت هر مضمون، حاضری ۳۰ روزه، رتبه در صنف، پیوند والدین).
- گزارش معلم هوشمند — از دادهٔ واقعی (نمرات، حاضری، روند) محاسبه می‌شود.
- تعلیق/فعال‌سازی/حذف با ذکر دلیل، و ارسال کد بازیابی رمز به ایمیل شاگرد.

Endpointهای جدید (زیر `/api/v1/admin`): `GET /students`, `GET /students/:id`,
`GET /students/:id/ai-report`, `PATCH /students/:id/status`,
`POST /students/:id/password-reset-link`.

### ۲) حافظهٔ جمعی (فید اجتماعی)
قبلاً فقط روی حافظهٔ همان دستگاه بود؛ حالا پست‌ها و کامنت‌ها روی سرور ذخیره و
بین همهٔ کاربران به‌اشتراک گذاشته می‌شوند. هویت نویسنده از JWT گرفته می‌شود
(ضد جعل هویت)؛ ویرایش/حذف فقط برای صاحب پست یا مدیر.

Endpointها (زیر `/api/v1/memory`): `GET/POST /posts`, `PATCH/DELETE /posts/:id`,
`POST /posts/:id/reactions`, `GET/POST /posts/:id/comments`, `DELETE /comments/:id`.

### ۳) کتابخانهٔ نصاب (پایگاه دانش معلم هوشمند)
متن استخراج‌شدهٔ کتاب‌های درسی حالا روی سرور ذخیره می‌شود تا بین همهٔ کاربران و
معلم هوشمند مشترک و ماندگار باشد.

Endpointها (زیر `/api/v1/curriculum-library`): `GET /books`,
`GET /subjects/:subjectId/books`, `POST /books` (مدیر), `DELETE /books/:id` (مدیر).

## مرحلهٔ لازم: اعمال Migrationهای جدید

```bash
cd backend
npx wrangler d1 execute afghan_girls_school_db --remote --file=./migrations/0014_collective_memory.sql
npx wrangler d1 execute afghan_girls_school_db --remote --file=./migrations/0015_curriculum_library.sql
# (اگر هنوز نکرده‌اید، 0013 هم لازم است — تأیید ایمیل/عکس پروفایل)
npx wrangler d1 execute afghan_girls_school_db --remote --file=./migrations/0013_email_verification.sql
```

سپس استقرار مجدد:

```bash
npx wrangler deploy
```

> نکته: `wrangler deploy` کد TypeScript را هنگام Build اعتبارسنجی می‌کند؛ اگر
> خطایی بود همان‌جا گزارش می‌شود.

هیچ تغییری در سمت اپ Flutter برای این سه بخش لازم نیست جز ساخت مجدد معمول
(همگی به‌صورت خودکار با سوییچ `kUseLiveBackend` که پیش‌فرض روشن است به سرور
وصل می‌شوند).

## بخشی که هنوز وصل نشده: «آکادمی»

بخش «آکادمی» (سیستم موازی امتحانات/بانک سؤال/تصحیح با دادهٔ نمونه) عمداً در این
مرحله به سرور وصل نشد، به دو دلیل مهم:

1. **ساختار متفاوت:** برخلاف بقیهٔ بخش‌ها که الگوی تمیز DataSource/Repository
   دارند، آکادمی یک انبار درون‌حافظه‌ایِ همگام (synchronous singleton) با
   شاگردان نمونهٔ ثابت است که در ده‌ها صفحه مستقیم استفاده می‌شود. اتصال آن
   نیازمند بازنویسی گسترده و پرریسکِ همهٔ آن صفحات است.
2. **هم‌پوشانی:** آکادمی با بخش‌های «امتحانات»، «کتابخانه» و «CMS» که از قبل به
   سرور وصل‌اند هم‌پوشانی دارد. وصل‌کردن جداگانهٔ آن، دو منبع حقیقتِ موازی برای
   امتحانات می‌سازد که خطر ناسازگاری داده دارد.

**پیشنهاد:** به‌جای اتصال جداگانهٔ آکادمی، آن را با سیستم «امتحانات» موجود
یکپارچه کنیم (یک منبع حقیقت روی سرور). این یک کار مجزا و برنامه‌ریزی‌شده است.
